\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
Artificial Intelligence has penetrated numerous sectors, fundamentally altering how operations are executed. However, in sensitive domains like finance, the "black box" nature of complex AI models creates a barrier to trust and regulatory compliance. Explainable AI (XAI) addresses this by providing human-understandable justifications for AI decisions. This project, titled "Financial Explainable AI Chatbot," proposes and develops an interactive system that leverages machine learning to yield financial insights while prioritizing transparency. The chatbot integrates real-time information retrieval methods, custom financial calculation tools (Simple Interest, Compound Interest, SIP, and FD computations), and an Explainable AI engine specifically dedicated to loan eligibility scoring. Powered by a decoupled architecture comprising a Flask backend and a Streamlit frontend, the system orchestrates user queries using intent routing and returns both computed answers and the logic behind them. Our loan engine features dual-mode functionality: predictive capability via a pre-trained machine learning model and a robust fallback mechanism utilizing a transparent, rule-based approach when models are unavailable. The user interacts through a modern natural language AI chat interface, breaking the barrier of traditional, clunky forms and rigid web portals. This thesis details the architectural design, algorithmic implementation, mathematical formulations, workflow architectures, and evaluation of the system. The provided system diagrams showcase the event loop driven UI orchestrating calls to a centralized routing engine. Overall, the system acts as a comprehensive demonstration of fostering user trust in financial technologies through interactive XAI interfaces.

\chapter{Introduction}

\section{Background}
The explosion of Artificial Intelligence (AI) algorithms in finance has revolutionized aspects of trading, risk assessment, fraud detection, and customer support. Institutions utilize powerful machine learning (ML) models to assess creditworthiness in milliseconds, optimizing operations that formally took days to manually compute. In previous decades, bank officers acted as the primary mediators evaluating a customer's loan constraints. Today, advanced statistical algorithms map non-linear relationships to generate predictions.

However, the models achieving the highest accuracy---like deep neural networks, Random Forests, and Gradient Boosting algorithms---often act as opaque "black boxes." When an end user engages with these systems, they provide inputs but cannot observe the inner calculation mechanisms. A user denied a loan receives a swift rejection without knowing whether it was due to a faulty input, minor credit history issues, or systemic algorithmic bias. Explainable AI (XAI) focuses on reversing this dilemma. XAI seeks to make AI models transparent, offering human-readable justifications behind prediction outcomes that end-users, branch officers, and regulatory bodies can understand. Incorporating this transparency into a conversational interface fundamentally shifts the user experience from confronting an opaque system to participating in a fluid, informative dialogue.

\section{Problem Statement}
Modern financial institutions deploy highly sophisticated, yet highly convoluted machine learning models that are frequently incomprehensible to the end users they impact.
\begin{enumerate}
    \item \textbf{Opacity in Decision Making:} When a user requests to check their loan eligibility, a generic "approved" or "rejected" outcome causes frustration and mistrust. They have no actionable feedback to improve their credit profile.
    \item \textbf{Fragmented Tools:} Legacy financial systems usually require users to navigate disjointed, complex forms for basic calculations (such as SIP returns or Compound Interest) or swap between independent brokerage apps to check real-time stock prices.
    \item \textbf{Lack of Conversational Fluency:} Applications lack the linguistic flexibility of human-to-human advisors, relying purely on rigid parameters. 
\end{enumerate}
Summarily, there is an evident lack of unified, intuitive conversational interfaces where users can effortlessly obtain financial insights combined with machine learning outputs that they can simultaneously trust through explicitly formatted, transparent explanations.

\section{Objectives}
The primary objective of this project is to architect, develop, and evaluate an Explainable AI Chatbot uniquely tailored for personal retail finance. The core objectives include:
\begin{itemize}

    \item \textbf{Provide Justified Outputs:} Guaranteeing verbose, explainable text outputs outlining exactly why the machine learning system or rule heuristical system made a specific financial decision.
\end{itemize}

\chapter{Literature Review}

\section{The Proliferation of Machine Learning in Finance}
Artificial Intelligence has been applied in finance for decades. Beginning with simple expert systems (such as the MYCIN methodologies applied to fraud detection) in the 1980s, the paradigm shifted toward rule-based algorithmic trading in the 2000s, and recently progressed drastically into Deep Learning applications natively embedded on consumer mobile applications. 
Machine learning systems leverage vast data lakes---comprising transaction histories, macroeconomic indicators, and alternative datasets like geolocation---to uncover subtle statistical patterns. Papers published by institutions such as JP Morgan Chase highlight that ensemble methods (e.g., XGBoost, Random Forests) significantly outperform logistic regression in credit default swaps. Yet, an enduring caveat to ML deployment in high-stakes fields like credit issuance is that extreme algorithmic complexity correlates directly to an extreme lack of human interpretability.

\section{The Crisis of the Black Box and Regulation}
A model acting as a "Black Box" implies that although inputs and outputs are visible, the intermediary transfer functions (the millions of intertwined weights and biases in a neural network, or the thousands of splitting parameters in an ensemble tree) remain profoundly indecipherable to human analysts. 
If an applicant uses a modern banking tool and is denied a mortgage, they have the right to question the decision. The advent of intense global regulatory frameworks---most notably the European Union's General Data Protection Regulation (GDPR) enacted in 2018---fundamentally altered algorithmic deployment by including clauses demanding a "Right to Explanation" for any fully automated decisions causing legal or significant consequences to the data subject. Thus, financial technology engineers are compelled to prioritize Explainable AI.

\section{Explainable AI (XAI) Paradigms}
Explainable AI (XAI) bridges the gap between deep mathematical algorithms and human comprehension. Several leading techniques dominate the academic landscape:
\begin{itemize}
    \item \textbf{LIME (Local Interpretable Model-Agnostic Explanations):} Proposed by Ribeiro et al. (2016), this technique perturbs input parameters slightly to witness how a generalized black-box model's output shifts, fitting a very simple, interpretable linear schema around a singular prediction to approximate why the model reached that specific conclusion.
    \item \textbf{SHAP (SHapley Additive exPlanations):} A game-theoretic approach introduced by Lundberg and Lee (2017). SHAP calculates the marginal contribution of each input feature across all possible combinations of features. It grants a mathematically sound interpretation of how much "credit score" or "monthly income" truly tipped the scale for a model.
    \item \textbf{White-Box Heuristics:} Developing highly explicit, rule-based algorithmic boundaries. This mechanism guarantees 100\% interpretability, sacrificing minor statistical nuances to achieve regulatory invulnerability.
\end{itemize}
This project heavily revolves around implementing systemic transparency natively mimicking XAI logic explicitly to users, ensuring predictions and calculation responses enumerate standard deviations, thresholds, and limits.

\section{Evolution of Conversational Interfaces}
Chatbots have moved drastically away from the state-machine and rigid decision trees utilized over the last two decades. While many chatbots employ Natural Language Processing (NLP) like Support Vector Machines mapping TF-IDF vectors, the introduction of Transformers has driven conversational UI expectations exponentially high. This thesis explores a distinct UI pattern utilizing Streamlit's innate reactive UI loops. While the system operates by routing hard-coded intents, the user interacts within a universally familiar messaging cadence, granting high fidelity interactions.

\chapter{System Design and Architecture}

\section{Design Paradigm Overview}
The system architecture was formulated utilizing a Microservices-inspired decoupled paradigm. By separating the user interface completely from the processing engine, and isolating calculating elements independently within the backend, the project maximizes modularity. This structure effectively handles concurrency asynchronously and allows singular components to fail (e.g., if the primary machine learning model `.pkl` file becomes corrupted) without causing a catastrophic halt to the global system.

\begin{itemize}
    \item \textbf{Streamlit UI:} The user types natural language into a chat input. The system renders real-time conversational logs using \texttt{st.chat\_message}.
    \item \textbf{Flask API Endpoint:} When a message is entered, the UI initiates a POST request to \texttt{127.0.0.1:5000/chat}. The Flask router sanitizes the JSON body (\texttt{\{"message": "string"\}}).

    \item \textbf{Specific Dispatchers:} Depending on the evaluated string, Python routes the properties parameters (like Principal amounts or stock ticker symbols) to native logic modules which compute exact results. 
\end{itemize}

\section{State Management}
One of the largest challenges facing stateless protocols (HTTP communication) coupled with AI conversation is memory retention. Streamlit manages local session states utilizing the \texttt{st.session\_state} global dictionary array. Every interaction is appended globally, keeping physical conversational rendering completely preserved upon frontend redeploys during dynamic interactions.

\chapter{Methodology and Implementation details}

\section{Mathematics of Financial Logic}
Our core calculation methodology strictly adheres to canonical banking formulations. Integrating these directly ensures high precision independent of external calculation servers.

\subsection{Simple and Compound Interest}
Simple Interest ($SI$) evaluation:
\begin{equation}
SI = \frac{P \times R \times T}{100}
\end{equation}
Where $P$ is principal, $R$ is annual rate, and $T$ is time in years.
Compound Interest ($CI$) evaluation:
\begin{equation}
A = P \left(1 + \frac{R}{n \times 100}\right)^{n \times T}
\end{equation}
Where $n$ represents the compounding frequency (e.g., 4 for quarterly).

\subsection{Systematic Investment Plan (SIP)}
Estimating future yields of continuous systematic investments leverages the future value of an annuity formulation:
\begin{equation}
FV = P \times \frac{(1 + i)^n - 1}{i} \times (1 + i)
\end{equation}
Where $P$ is the monthly contribution, $n$ is total months ($Years \times 12$), and $i$ represents the monthly fractional rate ($R / 12 / 100$).

\section{Explainable AI Loan Fallback Mechanism}
To demonstrate Explainability, calculating variables natively without machine learning allows complete control over the explanation strings generated. The fallback methodology evaluates a core set of features: Credit Score, Income, Debt, Loan Amount, Terms, and Duration.
The Debt-to-Income ($DTI$) metric is arguably the most sensitive predictor globally in personal issuance.
\begin{equation}
DTI\_Ratio = \frac{Monthly\_Debt\_Payments + (\frac{Loan\_Amount \times Estimated\_Interest}{12})}{Monthly\_Income}
\end{equation}
The algorithm dictates:
\begin{itemize}
    \item \textbf{Credit Criterion:} Automatically flags accounts displaying Credit $< 650$.
    \item \textbf{DTI Criterion:} Automatically flags profiles exceeding a stringent barrier of $40\%$.
\end{itemize}
Because these thresholds are hardcoded, the XAI mapping directly appends qualitative string explanations: \textit{"Credit profile displays significant risk (Score = 610) against our strict baseline of 650. DTI remains healthy (0.24)."}.

\section{Implementation Technology Stack}
The implementation process relied on highly active open-source repositories to fulfill backend operations seamlessly in a Linux environment.
\begin{itemize}
    \item \textbf{Python 3.10+}: Foundational language.
    \item \textbf{Streamlit}: Frontend engine. Exclusively utilizing \texttt{st.chat\_message()} interfaces. 
    \item \textbf{Flask}: Lightweight RESTful engine, chosen specifically over Django due to minimizing boilerplate code and rapid asynchronous JSON streaming setups.
    \item \textbf{yfinance}: The default endpoint scraper to acquire immediate quote execution via the Yahoo Finance API endpoints, bypassing heavy API keys.
    \item \textbf{scikit-learn \& pandas}: Heavy mathematical implementations handling dataframe management and potential Random Forest/Logistic Regression unpickling pipelines.
\end{itemize}

\section{Environment Security and Execution}
All executions require isolated virtual environments instantiated via \texttt{python -m venv .venv}. Execution dependencies reside securely inside \texttt{requirements.txt}. Cross-Origin Resource Sharing (CORS) implementations inside Flask are deliberately relaxed for internal local host access but configured to restrict unauthorized hostnames. Future setups support containerization via Docker using the provided \texttt{Dockerfile} ensuring cross-OS compatibility.

While functionally robust, certain performance disruptions exist relying on external web libraries. Fallback models (rule-based) are completely static, avoiding downtime entirely but lacking sophisticated insight into nuanced patterns mapped primarily by ML frameworks.

    \item \textbf{High-Frequency Orchestration Frameworks:} Modifying backend execution engines to utilize \texttt{FastAPI} with async workers instead of \texttt{Flask} guarantees superior execution density in severe traffic simulations.
\end{itemize}

import math

def calculate_simple_interest(principal: float, rate: float, time_years: float):
    si = (principal * rate * time_years) / 100
    total = principal + si
    return f"Simple Interest evaluated to: {si:.2f}\nTotal amount is: {total:.2f}"

def calculate_compound_interest(principal: float, rate: float, time_years: float, frequency: int = 1):
    amount = principal * math.pow((1 + (rate / (100 * frequency))), (frequency * time_years))
    ci = amount - principal
    return f"Compound Interest accumulated: {ci:.2f}\nTotal evaluated amount is: {amount:.2f}"

def calculate_sip(monthly_investment: float, annual_rate: float, time_years: float):
    """
    Standard formula for Systematic Investment Plan future evaluations.
    Takes annual rate, breaks it to monthly indexing.
    """
    n = time_years * 12
    i = annual_rate / 12 / 100
    
    future_value = monthly_investment * ( (math.pow(1 + i, n) - 1) / i ) * (1 + i)
    total_invested = monthly_investment * n
    wealth_gain = future_value - total_invested
    
    return f"**SIP Projection**: Over {time_years} years, your total execution investment equals: {total_invested:.2f}. Total generated wealth gain estimates to: {wealth_gain:.2f}. Projected absolute value: {future_value:.2f}"

        income = features_dict.get('monthly_income', 1)

        else:
            reasons.append(f"Credit profile is fundamentally strong ({score}).")

        else:

def attempt_load_model(model_path="models/model.pkl"):
    """Validates existence of Pickled Machine Learning structures."""

        try:
            with open(model_path, 'rb') as f:

        except Exception as e:

            return None, False
    return None, False
